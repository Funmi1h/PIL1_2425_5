L'organisation du travail s'est essentiellement basé sur les compétences dont disposent réellement chacun. Une maquette de l'application a été réalisé les deux premiers jours juste pour guider sur la réalisation du front-end. Le développement de ce dernier a été réalisé par quatre membres. Quant au back-end il a été réalisé par deux personnes.

Nous avons constitué 3 sous-groupes :

  • Equipe Front-end: Chargée de la rédaction du code source de l’interface web, côté client
  • Equipe Back-end: Chargée du développement de l'architecture serveur du projet ainsi que de la structure de la base de données
  • Equipe multimedia: Chargée de realisée une maquette design du site avec Figma

Technologies utilisées

  • HTML5, CSS3 et JavaScript pour le front-end de l'application.
  • Django pour la gestion des utilisateurs et la mise en correspondance des passagers et des conducteurs.
  • Leaflet.js et Nominatim pour le géocodage des adresses
  • Django Channels pour activer la communication en temps réel via WebSockets
  • WebSockets pour les échanges instantanés (notifications)
  • Daphne (serveur ASGI) pour exécuter l'application de maniere asynchrone
  • Utilisation de MySQL pour la gestion de la base de données.
  • Utilisation de Git pour le versionnement du projet
